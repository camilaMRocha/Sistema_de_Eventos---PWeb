# Sistema_de_Eventos---PWeb

* Nome do projeto -
* Tema escolhido - Sistema de eventos
* Objetivo do sistema -
* Tecnologias que serão utilizadas durante o semestre - Python
* Breve descrição do funcionamento do sistema – O sistema tem como objetivo facilitar a organização de eventos como casamentos, aniversários, festas e formaturas, conectando clientes a espaços, estruturas e profissionais (buffet, decoração, fotografia, som, etc.) necessários para a realização do evento. Por meio da plataforma, o usuário poderá pesquisar locais e serviços disponíveis, verificar datas, orçamentos e realizar a contratação de forma centralizada, acompanhando todo o planejamento até a data do evento
* Realizar o levantamento dos requisitos do sistema, descrevendo:

  | Código | Descrição |
  | :--- | ---: |
  | RF01 |	O sistema deve permitir o cadastro de usuários (clientes). |
  | RF02 |	O sistema deve permitir o login e autenticação dos usuários. |
  | RF03 |	O sistema deve permitir o cadastro de fornecedores/profissionais (buffet, decoração, fotografia, som, etc.). |
  | RF04 |	O sistema deve permitir o cadastro de locais/espaços para eventos. |
  | RF05 |	O sistema deve permitir que o cliente pesquise locais disponíveis por data, tipo de evento e localização. |
  | RF06 |	O sistema deve permitir que o cliente pesquise profissionais/serviços por categoria (ex: fotografia, decoração). |
  | RF07 |	O sistema deve permitir a verificação de disponibilidade de data para locais e profissionais. |
  | RF08 |	O sistema deve permitir que o cliente monte um "pacote" de evento combinando local + serviços. |
  | RF09 |	O sistema deve calcular o orçamento estimado do evento com base nos itens escolhidos. |
  | RF10 |	O sistema deve permitir a contratação (reserva) do local e dos profissionais escolhidos. |
  | RF11 |	O sistema deve gerar um contrato ou comprovante de reserva. |
  | RF12 |	O sistema deve permitir o pagamento (ou registro de pagamento) da reserva. |
  | RF13 |	O sistema deve enviar notificações/lembretes sobre datas e prazos do evento. |
  | RF14 |	O sistema deve permitir que o cliente acompanhe o status do planejamento do evento (ex: confirmado, pendente, concluído). |
  | RF15 |	O sistema deve permitir avaliação e comentários sobre locais e profissionais após o evento. |
  | RF16 |	O sistema deve permitir que o administrador gerencie cadastros de locais e profissionais. |
  | RF17 |	O sistema deve permitir o cancelamento de uma reserva, respeitando regras de prazo. |
  | RF18 |	O sistema deve permitir a edição dos dados do evento antes da confirmação final. |

  | Código | Descrição |
  | :--- | ---: |
  | RNF01 |	O sistema deve ser desenvolvido em Python. |
  | RNF02 |	O sistema deve ter interface simples e intuitiva para usuários sem conhecimento técnico. |
  | RNF03 |	O sistema deve responder às ações do usuário em tempo aceitável (ex: até 3 segundos). |
  | RNF04 |	O sistema deve garantir a segurança dos dados pessoais e de pagamento dos usuários. |
  | RNF05 |	O sistema deve estar disponível para uso a maior parte do tempo (alta disponibilidade). |
  | RNF06 |	O sistema deve ser escalável para suportar aumento de usuários e cadastros. |
  | RNF07 |	O sistema deve manter um padrão visual consistente em todas as telas. |
  | RNF08 |	O código deve seguir boas práticas de organização para facilitar manutenção futura. |
